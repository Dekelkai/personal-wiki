# ✍️ 写作与输出知识地图

写作将研究、项目和工程实践转化为论文、报告和技术文章。内容应以可核验的来源、实验和项目记录为基础。

## 🎓 学术论文与投稿

学术写作需要围绕研究问题、方法、实验和局限形成清晰论证。投稿版本、格式要求与审稿意见应与对应的证据材料保持一致。

## 📖 综述与学位论文

研究综述按照问题、方法路线与评测条件组织文献。学位论文则需要在综述、方法、实验和贡献之间保持完整的叙事关系。

## 📝 项目文档与技术报告

项目文档说明目标、系统设计、实验验证和技术决策；技术报告面向具体读者选择合适的细节和证据。

## 🌐 技术博客

技术博客适合呈现可复用的工程经验、工具实践和研究方法解释。公开内容不应包含未核验结论、私人笔记或敏感配置。

## 🧾 Markdown、LaTeX 与 Overleaf

Markdown 适合轻量协作与版本管理，LaTeX 和 Overleaf 适合公式、引用和版式要求较高的文档。

## 📊 图表与可视化

图表应服务于论证与比较，清楚说明数据来源、指标、坐标和适用范围，避免用视觉效果替代结论。
