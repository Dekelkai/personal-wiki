## 💻 编程语言

Go 适合网络服务与并发程序，Python 用于科研计算和自动化脚本；C/C++、CMake 与编译链接问题构成理解原生工程的重要基础。

## 🖥️ 操作系统与开发环境

Linux 与 Windows 页面记录文件、权限、进程、服务、网络和远程开发等主题。环境配置需要同时关注版本、路径、依赖和权限。

## 📦 容器与部署

Docker 将应用及其依赖封装为可复用环境。镜像、容器、数据卷、网络和 Compose 构成常见的开发与部署单元。

## 🌿 版本控制与协作

Git 用于追踪代码和文档的演变。提交、分支、合并与回滚应与测试、审查和发布流程结合使用。

## 🧩 后端开发

后端系统通常围绕接口设计、数据与状态、认证授权、日志和可观测性展开；服务边界与错误模型需要在实现前明确。

## 🌐 计算机网络

TCP/IP、HTTP、DNS、路由、代理和 SSH 构成网络排查与服务通信的基础。定位问题时可从地址、端口、名称解析和应用日志逐层检查。
