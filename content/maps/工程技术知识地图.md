- [[Linux|Linux]]：文件、权限、进程、服务、网络与远程开发。
- [[Windows|Windows]]：系统管理、PowerShell 与开发环境。

- [[计算机网络|计算机网络]]：TCP/IP、HTTP、DNS、代理与 SSH。

## 🧩 核心分区

### 编程语言

以 Go 和 Python 为当前主要入口，C/C++ 内容在形成稳定笔记后再建设页面。

- Go 后端与网络编程
- Python 科研与自动化脚本
- C/C++ 编译、链接和 CMake

### 系统与开发环境

围绕 Linux、Windows 和跨平台环境配置，记录版本、路径、权限与依赖关系。

- 命令行与环境变量
- 进程、服务和文件权限
- 远程开发与工作站配置

### 容器与版本控制

Docker 固化运行环境，Git 记录内容和代码变化；两者都需要明确数据与凭据边界。

- 镜像和 Compose 工作流
- 分支、提交、合并和回滚
- 构建、测试与发布前检查

### 后端与网络

从接口、状态和可观测性出发理解后端系统，并使用分层网络模型定位连接问题。

- REST API 与 WebSocket
- 数据库、缓存和消息机制
- HTTP、DNS、路由、代理与 SSH

## 🔥 当前重点

1. 统一 Windows、Linux、Python 和 Go 环境记录方式。
2. 为科研项目建立可复现的 Docker 与依赖说明。
3. 整理常用 Git 操作、提交规范和知识库版本流程。
4. 将故障记录从零散命令升级为可复现的问题排查页面。
5. 沉淀 Go 后端项目的接口、测试和部署实践。
