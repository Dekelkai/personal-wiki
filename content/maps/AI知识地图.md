- [[MCP|MCP]]：Client、Server、工具、资源与安全考虑。

## 🧩 核心分区

### 模型与调用

关注大语言模型、多模态模型、上下文窗口、Token、缓存、模型路由与调用成本。

- 模型能力与局限
- 多模态理解与视觉语言模型
- Token、缓存和请求管理
- 模型评测与供应商差异

### AI Agent

关注目标分解、工具调用、上下文管理、记忆、权限和验证，而不是只记录产品功能。

- 单 Agent 工作循环
- 子任务与多 Agent 协作
- 项目级上下文与长期记忆
- 权限、沙箱和人工确认

### 工具与协议

Codex CLI、Claude Code 和 MCP 是当前主要入口。`AGENTS.md` 是根目录治理文件，正文中以代码形式引用，不作为内容 Wiki Link。

- 编程 Agent 工具
- MCP 工具、资源与配置
- 知识管理与科研工具
- 工具故障排查

### 工作流与提示词

将 AI 能力放入科研、开发和知识库维护流程，用可核验结果约束模型输出。

- 提示词结构与版本管理
- 科研阅读、实验和写作流程
- 开发实现、测试和代码审查
- 知识库整理、链接检查与变更摘要

## 🔥 当前重点

1. 建立 Codex CLI 与 Claude Code 的可复用项目协作规范。
2. 明确 Agent 的文件权限、外部操作和敏感信息边界。
3. 记录 MCP Server 的来源、能力范围和故障排查过程。
4. 评估上下文压缩、缓存和模型调用对任务质量的影响。
5. 将 AI 工作流连接到真实科研和开发任务，而非孤立技巧。
