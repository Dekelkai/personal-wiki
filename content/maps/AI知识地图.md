这里汇集模型、智能体、工具协议、提示词和工作流相关内容，关注它们在科研与软件开发中的实际使用。

## 🧠 模型与基础

大语言模型和多模态模型通过上下文、Token 和工具调用处理任务。模型能力、成本、缓存与评测需要放在具体任务中理解。

## 🤖 智能体与编程工具

智能体将模型、上下文、工具和反馈循环组合起来。Codex CLI 与 Claude Code 聚焦项目协作、代码处理和受控的文件操作。

## 🔌 MCP 与工具集成

MCP 为工具、资源和提示能力提供统一的连接方式。配置工具时应明确来源、权限范围和可访问的数据。

## 💬 提示词与上下文

提示词需要说明目标、约束、输入、输出和验收标准；上下文应区分任务必需信息、长期规范与不可信资料。

## 🔄 科研与开发工作流

[[AI辅助科研|AI 辅助科研]] 讨论文献、实验与写作中的核验边界，[[AI辅助开发|AI 辅助开发]] 连接需求、实现、测试与交付过程。

## 🔐 安全、权限与成本

AI 工具应在最小权限下运行，避免敏感信息进入上下文或日志。Token 消耗、缓存命中和调用稳定性是评估工作流的常见维度。

## 🩺 常见问题

模型输出、工具调用和上下文限制都需要通过可复现的输入、日志和验证结果定位，而不是仅依据单次对话判断。
