# 📏 视觉定位误差指标

视觉定位任务可能输出检索候选、二维位置、三维位置、相机姿态或连续轨迹，因此不存在一个能够替代全部评价的单一指标。比较结果前必须先确认任务输出、坐标系、对齐方式、单位、失败样本处理和聚合规则。

## 指标定义

### 单帧位置误差

对于估计位置 $\hat{\mathbf{p}}_i$ 和真值位置 $\mathbf{p}_i$，欧氏位置误差可写为：

$$
e_i = \lVert \hat{\mathbf{p}}_i - \mathbf{p}_i \rVert_2
$$

二维任务只计算平面坐标，三维任务包含高度。经纬度不能在未投影或未说明地球模型时直接按普通米制坐标相减。

### 均方根误差

对 $N$ 个位置误差，RMSE 为：

$$
\operatorname{RMSE} = \sqrt{\frac{1}{N}\sum_{i=1}^{N} e_i^2}
$$

RMSE 对大误差更敏感。它应与中位数、分位数、最大值、成功率和失败样本数一起报告，不能只保留一个平均数字。

### 绝对位姿或轨迹误差

Absolute Pose Error（APE）直接比较对应时刻的估计位姿和参考位姿，常用于衡量轨迹的全局一致性。Absolute Trajectory Error（ATE）在许多工作中指对整条轨迹的绝对误差统计，但具体实现可能包含不同对齐方式。

### 相对位姿误差

Relative Pose Error（RPE）比较两个时刻之间的相对运动，可用于分析局部精度和漂移。评价需要记录帧间隔或距离间隔，以及计算的是平移还是旋转关系。

### 旋转误差

旋转误差衡量估计姿态与真值姿态之间的角度差，常以度表示。必须说明姿态表示、坐标系方向以及比较的是完整旋转、航向角还是其他分量。

### Recall@K

Recall@K 用于地点识别或图像检索：当正确地图候选出现在前 K 个结果中时记为成功。它反映候选召回能力，不等同于最终米制位置或 6-DoF 位姿精度。

### 运行速度

FPS、Hz、单帧毫秒数和端到端延迟衡量计算性能，不是定位精度。报告时需要说明硬件、批量大小、是否包含地图检索、预处理和后端优化。

## 单位与方向

| 指标 | 常见单位 | 方向 |
| --- | --- | --- |
| 二维或三维位置误差 | m、cm | 越低越好 |
| APE/ATE 平移误差 | m | 越低越好 |
| RPE 平移漂移 | m、m/m、% | 越低越好 |
| 旋转误差 | °、°/m、°/s | 越低越好 |
| Recall@K | % | 越高越好 |
| 成功率 | % | 越高越好 |
| 延迟 | ms | 越低越好 |
| FPS 或 Hz | frame/s、Hz | 越高越好，但需要结合精度 |

## 聚合方式

建议同时记录：

- 有效样本数和失败样本数；
- 均值、中位数、RMSE、标准差和关键分位数；
- 每条轨迹结果，而不是只给所有轨迹混合后的总值；
- 不同场景、高度、视角和时间条件下的分组结果；
- 成功阈值及其依据。

如果失败样本被排除，必须同时报告失败率，否则误差统计可能只描述成功子集。

## 适用任务

- [[AerialVL]]：可同时使用 Recall@K、位置误差、序列成功率和长航线稳定性指标；
- [[OrthoLoC]]：适合报告平移误差、旋转误差、定位成功率和相机标定误差；
- [[JointLoc数据集|JointLoc 数据集]]：适合报告世界坐标轨迹的 APE/ATE、局部漂移、位置误差统计和运行速度。

具体使用哪些指标，应以数据集官方协议和研究问题为准。

## 可比条件

只有以下条件基本一致时，数值比较才有意义：

- 使用相同数据版本和测试划分；
- 查询、地图和真值预处理一致；
- 坐标系、单位和时间关联一致；
- 使用相同的轨迹对齐规则；
- 失败样本处理和成功阈值一致；
- 聚合方式和轨迹权重一致；
- 运行速度使用可比较的硬件和计时范围。

`evo` 支持 SE(3)、Sim(3)、尺度和原点等不同对齐方式。经过 Sim(3) 尺度对齐的单目 SLAM 轨迹误差，不能与不允许任何后验对齐的世界坐标绝对定位误差直接比较。

## 常见误用

- 用 Recall@1 代替最终地理位置精度；
- 将地点数量、图像数量和轨迹数量混为数据规模；
- 只报告成功样本的均值，不报告定位失败；
- 将经过最佳对齐的轨迹称为直接世界坐标输出；
- 把 RMSE、均值和中位数当作可以互换的统计；
- 把论文作者报告结果写成用户复现结果；
- 比较不同数据划分、地图分辨率或坐标系下的数字；
- 把 FPS 当作端到端系统实时性而不说明计时范围。
