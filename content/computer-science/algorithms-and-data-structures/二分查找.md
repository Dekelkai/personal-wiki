# 🔎 二分查找

二分查找在具有单调性的搜索空间中，每次排除约一半候选范围。它不仅用于查找已排序数组中的某个值，也可用于查找“第一个满足条件的位置”。

## 前提条件

- 候选区间可以随机访问；
- 数据已排序，或判定条件在区间上具有单调性；
- 比较规则和排序规则一致。

如果数据尚未排序，为单次查找先执行 `O(n log n)` 排序通常不如直接 `O(n)` 线性扫描。

## 区间与不变式

使用左闭右开区间 `[left, right)` 可以统一空区间和边界处理：

- 搜索区间中的元素索引始终满足 `left <= i < right`；
- 区间长度是 `right - left`；
- 当 `left == right` 时区间为空。

## 查找确切值

```go
func BinarySearch(values []int, target int) (int, bool) {
	left, right := 0, len(values)

	for left < right {
		mid := left + (right-left)/2
		if values[mid] < target {
			left = mid + 1
		} else {
			right = mid
		}
	}

	if left < len(values) && values[left] == target {
		return left, true
	}
	return left, false
}
```

该实现首先找到第一个大于等于 `target` 的位置，再判断该位置是否真正等于目标值。如果目标不存在，返回值仍是保持有序时可插入的位置。

## 正确性要点

循环中始终保持：

- `left` 左侧的元素都小于 `target`；
- `right` 及其右侧的候选位置不会比答案更靠左；
- 每次迭代都使 `[left, right)` 严格缩小。

循环结束时 `left == right`，该位置就是第一个不小于目标值的索引。

## 复杂度

- 时间复杂度：`O(log n)`；
- 额外空间：迭代实现为 `O(1)`；
- 如果输入不支持常数时间随机访问，实际成本需要重新分析。

复杂度记号见[[时间与空间复杂度]]。

## 常见变体

- 第一个大于等于目标的位置；
- 第一个严格大于目标的位置；
- 最后一个小于等于目标的位置；
- 对答案空间二分，例如寻找满足约束的最小容量、最大阈值或最早时刻。

## 常见错误

- 区间定义在循环中变化，导致越界或遗漏答案；
- 在闭区间写法中直接使用 `(left + right) / 2`，在其他语言或极端规模下可能溢出；
- 对降序数据仍使用升序比较规则；
- 只测试目标存在的情况，没有测试空数组、单元素、重复元素和两端边界。

## 测试样例

```go
tests := []struct {
	values []int
	target int
}{

	{[]int{3}, 3},
	{[]int{1, 3, 3, 5}, 3},
	{[]int{1, 3, 5}, 4},

```
