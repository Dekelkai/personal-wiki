# 🧩 数组与切片

数组用连续的索引组织同类型元素。Go 中的数组长度固定并且是类型的一部分；切片则是对某个底层数组连续区间的描述，更适合大多数可变长序列。

## Go 数组

```go
var fixed [3]int
fixed[0] = 10

values := [3]int{10, 20, 30}
inferred := [...]int{10, 20, 30}
```

- `[3]int` 和 `[4]int` 是不同类型；
- 数组赋值和函数传参会复制整个数组；
- 按索引读写元素的时间复杂度是 `O(1)`。

## Go 切片

```go
numbers := []int{10, 20, 30}
window := numbers[1:3]

fmt.Println(len(window)) // 2
fmt.Println(cap(window)) // 从起点到底层数组末尾的容量
```

切片可以理解为包含底层数组引用、长度和容量的描述值。多个切片可以共享同一底层数组。

```go
base := []int{1, 2, 3, 4}
part := base[1:3]
part[0] = 99

fmt.Println(base) // [1 99 3 4]
```

## 长度、容量与扩容

- `len(s)` 是切片当前可访问的元素数；
- `cap(s)` 是从切片起点到底层数组末尾可用的元素数；
- `append` 在容量足够时可以复用原底层数组，容量不足时会分配新数组并复制元素。

```go
items := make([]int, 0, 4)
items = append(items, 10)
items = append(items, 20, 30)
```

`append` 必须接收返回值，因为返回的切片可能已经引用新的底层数组。

## 复制与独立存储

需要独立副本时，应显式分配并复制：

```go
source := []int{1, 2, 3}
target := make([]int, len(source))
copy(target, source)
```

也可使用追加到 `nil` 切片的方式创建副本：

```go
target := append([]int(nil), source...)
```

## 常见操作与复杂度

| 操作 | 典型时间复杂度 | 说明 |
| --- | --- | --- |
| 按索引访问 | `O(1)` | 已知位置直接定位 |
| 按值查找 | `O(n)` | 未排序时需线性扫描 |
| 末尾追加 | 均摊 `O(1)` | 扩容时单次可为 `O(n)` |
| 中间插入或删除 | `O(n)` | 通常需要移动后续元素 |
| 复制 | `O(n)` | 按元素复制 |

复杂度记号的完整含义见[[时间与空间复杂度]]。

## 删除元素

保留顺序的删除需移动后续元素：

```go
func removeAt(values []int, index int) []int {
	copy(values[index:], values[index+1:])
	return values[:len(values)-1]
}
```

如果元素类型持有指针或大对象，可在缩短前把末尾位置设为零值，避免底层数组继续保留不需要的引用。

## 常见易错点

- 误以为子切片与原切片相互独立；
- 保留很小的子切片，却因共享底层数组而使大块内存无法回收；
- 忽略 `append` 后新旧切片可能不再共享存储；
- 在遍历切片时同时修改长度，导致跳过元素或越界；
- 不检查索引范围就执行切片操作。
