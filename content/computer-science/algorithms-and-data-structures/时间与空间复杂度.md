# ⏱️ 时间与空间复杂度

复杂度描述算法所需计算量或额外存储量如何随输入规模增长。它关注增长趋势，不直接等于某台机器上的具体运行时间或内存字节数。

## 输入规模

分析前首先定义 `n` 表示什么。它可以是数组长度、图的顶点数、边数、字符串长度或其他与问题相关的尺度。

图算法常同时使用 `V` 和 `E` 表示顶点数与边数，写成 `O(V + E)` 比简化成单一 `n` 更准确。

## 渐进记号

| 记号 | 含义 | 用途 |
| --- | --- | --- |
| `O(g(n))` | 渐进上界 | 描述增长速度不超过某个数量级 |
| `Ω(g(n))` | 渐进下界 | 描述至少需要的数量级 |
| `Θ(g(n))` | 渐进紧界 | 上界和下界属于同一数量级 |

工程文档中常用大 `O` 表示最坏情况的渐进上界，但严格来说大 `O` 本身不等于“最坏情况”。页面需另外说明分析的情况。

## 常见增长量级

| 复杂度 | 典型操作 | 规模增大时的特征 |
| --- | --- | --- |
| `O(1)` | 按索引访问数组 | 操作数与 `n` 无关 |
| `O(log n)` | [[二分查找]] | 每步按固定比例缩小搜索空间 |
| `O(n)` | 线性扫描 | 输入翻倍，主要工作量约翻倍 |
| `O(n log n)` | 常见比较排序 | 线性层工作与对数层次结合 |
| `O(n²)` | 双重遍历所有元素对 | 输入翻倍，工作量约增至四倍 |
| `O(2ⁿ)` | 枚举所有二元选择 | 只适合较小输入或经过强剪枝的问题 |

## 分析步骤

1. 定义输入规模；
2. 选择需要计数的基本操作；
3. 写出该操作的执行次数；
4. 保留主导项，忽略常数因子和低阶项；
5. 说明最好、平均、最坏或均摊情况。

例如，两个前后执行的线性循环是 `O(n) + O(n) = O(n)`；一个遍历 `n` 次的循环内再遍历 `n` 次，通常是 `O(n²)`。

## 空间复杂度

空间复杂度应明确是否只计算额外空间。递归函数的调用栈、新建的数组、哈希表和队列都属于额外空间。

- 原地交换数组元素可以是 `O(1)` 额外空间；
- 创建与输入等长的结果数组是 `O(n)` 额外空间；
- 深度为 `h` 的递归需要 `O(h)` 调用栈。

## 均摊复杂度

某些操作偶尔很贵，但一系列操作的平均成本较低。动态数组扩容是典型例子：单次扩容可能复制许多元素，但连续追加的均摊成本仍可视为 `O(1)`。

## 常见误区

- 只写 `O(n)` 却没有定义 `n`；
- 把平均情况当作最坏情况；
- 忽略排序、切片拷贝或哈希冲突的成本；
- 认为复杂度相同就意味着实际性能相同；
- 不说明额外空间是否包含输入和输出。
